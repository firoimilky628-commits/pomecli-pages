PomeCLI セットアップガイド

Last updated: 2026-06-23

初めて使う場合は、まず iPhone / iPad と Mac を同じWi-Fiにつないで試してください。Tailscale、VPS、ポート開放は、外出先から使いたい時だけ必要です。

必要なもの

  • PomeCLIを入れた iPhone / iPad。
  • 接続先の Mac または SSH サーバー。
  • SSHで使うユーザー名とパスワード。
  • 接続先に入っている tmux。

PomeCLIはSSHサーバーを提供しません。自分のMacやサーバーへ接続するためのアプリです。

最短セットアップ: 家のWi-FiでMacにつなぐ

1. Macでリモートログインをオンにする

Macで「システム設定」→「一般」→「共有」を開き、「リモートログイン」をオンにします。許可するユーザーを限定している場合は、自分のMacユーザーが許可されていることを確認してください。

2. Macのユーザー名とIPアドレスを確認する

Macの「ターミナル」を開いて、次を実行します。

whoami
ipconfig getifaddr en0

whoami の結果がPomeCLIに入れるユーザー名です。ipconfig getifaddr en0 の結果が、同じWi-Fi内で使うMacのIPアドレスです。例: 192.168.1.23

IPアドレスが表示されない場合は、MacのWi-Fiがオンか確認してください。有線LANや環境によっては ipconfig getifaddr en1 で表示されることもあります。

3. tmuxを確認する

PomeCLIはSSH接続後、tmuxセッションを一覧表示して操作します。Macのターミナルで次を実行します。

tmux -V

バージョンが表示されればOKです。command not found と出る場合は、Homebrewなどでtmuxをインストールしてください。

brew install tmux

4. tmuxセッションを作る

Mac側で、PomeCLIから選ぶためのtmuxセッションを作ります。

tmux new -s pome

この画面を開いたままでも、Control + b の後に d を押して一度離れても大丈夫です。

5. PomeCLIに接続情報を入力する

入力欄 入れるもの 例
MacのIPアドレスまたはホスト名 ipconfig getifaddr en0 の結果 192.168.1.23
Port 通常は22 22
Macのユーザー名 whoami の結果。Apple IDのメールアドレスではありません。 hiro
SSHパスワード Macにログインするときのパスワード。Apple IDのパスワードではありません。

入力後に「接続」を押し、tmuxセッション一覧で pome を選べば接続完了です。

外出先から使いたい場合

外から使うには、iPhone / iPad から接続先へ届くネットワーク経路が必要です。おすすめ順は次の通りです。

方法 向いている人 注意点
TailscaleなどのVPN 自宅Macを外出先から使いたい人。 iPhone / iPad と Mac の両方にVPNアプリの設定が必要です。
VPS / 公開SSHサーバー 常時動く開発環境がほしい人。 サーバー契約、SSH設定、セキュリティ管理が必要です。
自宅ルーターのポート開放 SSH、ファイアウォール、ルーター設定を理解している人。 初心者にはおすすめしません。SSHを直接インターネットへ公開するため、設定ミスのリスクがあります。

SSH鍵で高速接続する場合

最初はパスワード接続で動作確認してください。接続できることを確認した後、PomeCLIの「高速接続用の鍵を作成」を使うと、次回以降の接続を短い手順にできます。

  • PomeCLIで「高速接続用の鍵を作成」を押す。
  • 「Macへ登録するコマンドをコピー」を押す。
  • Macのターミナルへ貼り付けて実行する。
  • PomeCLIの「SSH鍵で高速接続」をオンにする。

うまく接続できない時

症状 確認すること
接続がタイムアウトする iPhone / iPad と Mac が同じWi-Fiか、VPNが接続済みか、IPアドレスが正しいかを確認します。
接続が拒否される Macのリモートログインがオンか、Portが22か、会社や学校のネットワークでSSHがブロックされていないかを確認します。
認証に失敗する ユーザー名は whoami の結果、パスワードはMacログイン用のパスワードです。
tmuxセッションが表示されない Mac側で tmux new -s pome を実行してから、PomeCLIで再接続します。
ホスト鍵の警告が出る Macやサーバーを作り直した、または別の接続先へ同じIPでつないでいる可能性があります。意図した接続先か確認してください。

English Summary

For the first setup, connect your iPhone / iPad and Mac to the same Wi-Fi network. Enable Remote Login on the Mac, get the Mac user name with whoami, get the local IP address with ipconfig getifaddr en0, make sure tmux is installed, then enter those values in PomeCLI.

Tailscale is not required for local Wi-Fi use. Use Tailscale or another VPN, a VPS, or a carefully managed public SSH setup only when you need to connect from outside your home network.
